spawn_agent iOS UI

Three UI surfaces for multi-agent sessions: workspace session list with parent-child grouping, a collapsible status bar in the parent's ChatView, and tap-to-navigate to any child.

Screen 1: Workspace Detail

Child sessions are visually nested under their parent with tree lines and indentation. The parent row shows a badge with the child count. Tapping any row navigates into that session's chat.

Design notes:
Collapsible: Disclosure triangle on parent rows toggles children visibility. Collapsed badge shows condensed status counts (⏳1 ✓1 ✗1). Default: expanded if any child is active, collapsed if all children are stopped.

Nested spawns: Children can spawn grandchildren — same tree pattern, deeper indent. "Extract token service" has its own disclosure triangle and one child ("Write token cache tests"). The tree line connector runs through all depth levels.

Non-parent rows: Standalone sessions have an empty spacer where the disclosure triangle would be, keeping text alignment consistent.

ChatView — Status Bar (Expanded)
Design note: The status bar sits between the chat timeline and the composer. Collapsed shows aggregate counts. Expanded reveals per-child rows with status dots, model, cost, and duration. Tapping a row navigates to that child's ChatView. The bar only appears when the current session has children.

Child ChatView — Full Session
Design note: The child session is a full ChatView — same rendering pipeline as any session. The "parent" breadcrumb is a tappable back link to the parent's chat. A subtle "Spawned by..." label gives context without taking up space. The child session has its own composer — you can interact with it independently.

Navigation Flow

Workspace Detail
├── Parent Session → ChatView (with status bar)
│ ├── status bar row tap → Child ChatView
│ │ └── parent breadcrumb ← back to Parent
│ └── spawn_agent tool row tap → Child ChatView
├── Child Session (directly) → ChatView
│ └── parent breadcrumb ← back to Parent
└── Standalone Session → ChatView (no status bar)
