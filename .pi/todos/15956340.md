## Context

Server exposes file access API (TODO-362ce018). iOS needs a view to browse
the agent's workspace and preview results.

Depends on: TODO-362ce018 (server file API)
Tracker: TODO-992ad1a6

## What to do

### A. FileService

New `ios/PiRemote/Core/Services/FileService.swift`:

- `listFiles(sessionId, path?)` → `[FileEntry]`
- `fileURL(sessionId, path)` → URL for content endpoint
- Model: `FileEntry { name, path, size, modified, isDir }`

Uses existing `APIClient` for authenticated requests.

### B. FileListView + FileRowView

New `ios/PiRemote/Features/Files/FileListView.swift`:

- Takes sessionId, optional subpath
- Shows directory listing sorted: dirs first, then files by modified desc
- FileRowView: icon (SF Symbol by type), name, size, modified time
- Tap directory → push FileListView for subdir
- Tap file → push FilePreviewView
- Pull to refresh
- Breadcrumb path in nav bar when in subdirectories
- SF Symbols: `folder` for dirs, `doc.text` for text, `photo` for images,
  `globe` for HTML, `tablecells` for CSV

### C. Integration into session detail

Add segmented control or tab to session detail screen:
```
[ Chat ]  [ Output ]
```

"Output" shows FileListView for the session's /work directory.
