## Summary

Create `src/gate.ts` — a `GateServer` class that manages per-session Unix sockets for the permission flow.

## Design

- Creates Unix socket per session at `/tmp/pi-remote-gate/<sessionId>.sock`
- Handles ndjson protocol: `guard_ready`, `gate_check`, `heartbeat`
- Uses `PolicyEngine` for evaluation
- When policy says "ask" → emits `approval_needed` event for server to forward to WebSocket
- In-memory `Map` for pending decisions with timeout cleanup
- Socket cleanup on session end

## API

```typescript
class GateServer extends EventEmitter {
  constructor(policy: PolicyEngine)
  createSessionSocket(sessionId: string, userId: string): string
  destroySessionSocket(sessionId: string): void
  resolveDecision(requestId: string, action: "allow" | "deny"): void
  // Events: "approval_needed", "guard_ready", "guard_lost"
}
```
