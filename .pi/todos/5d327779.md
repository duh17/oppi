## Summary

Replace auth.json copying with a host-side auth proxy. Switch containers to `--internal` network (no internet). Inspired by Gondolin's secret placeholder pattern.

## Design doc

`pi-remote/docs/auth-proxy-design.md` — contains Gondolin comparison, experiment results, source code trace, and implementation plan.

## Key pieces

1. `src/auth-proxy.ts` — HTTP reverse proxy, injects real credentials at request time (~150 LOC)
2. `src/sandbox.ts` — stub auth.json, rewrite models.json baseUrl, `--network pi-internal`
3. `src/sessions.ts` — register/remove sessions with proxy
4. Network: `container network create --internal --subnet 10.200.0.0/24 pi-internal`

## Validated claims (see design doc for details)

- models.json override-only path works for built-in providers (no apiKey/models needed)
- Custom headers from models.json are sent on every API request
- Apple container `--internal` network blocks all internet + DNS but allows host gateway
- auth.json policy rules are trivially bypassed (base64, globs, python, etc.)
- Pi's Anthropic SDK uses x-api-key for API keys, Bearer for OAuth (sk-ant-oat prefix check)

## Depends on

Nothing — can be done independently. Fetch skill will break on internal network (separate issue).
