## Summary (NNW §1.1)

Session transitions (load history → connect stream → request state) can race if triggered rapidly. Add a serialized async operation queue to ensure these steps execute in order.

## Why

NetNewsWire pattern: deterministic state transitions prevent subtle ordering bugs. Currently, navigating between sessions quickly could produce overlapping loads and stale data.

## Scope

- New utility: serialized async queue (Swift actor or `AsyncChannel`)
- Wire into `ChatView.connectToSession` flow
- Ensure `disconnect → load → connect → state` always runs in order
