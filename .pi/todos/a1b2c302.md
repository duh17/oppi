## Summary

Create `src/policy.ts` — a `PolicyEngine` class that evaluates tool calls against policy rules using glob matching.

## Design

- `evaluate(tool, input)` → `{ action: allow|ask|deny, reason, risk }`
- Glob matching against tool name + command/path (using `minimatch`)
- Three presets: `admin`, `standard`, `restricted`
- Pure logic — no I/O, no sockets, fully unit-testable

## API

```typescript
class PolicyEngine {
  constructor(preset: "admin" | "standard" | "restricted")
  evaluate(tool: string, input: Record<string, unknown>): PolicyDecision
}
```

## Scope

- New file: `pi-remote/src/policy.ts`
- New test: `pi-remote/tests/policy.test.ts`
- Add `minimatch` dependency
- 1-2 hour estimate
- v1 Step 2

## Acceptance

- All three presets have sensible defaults
- Known-safe tools (Read, Glob) auto-allow in standard preset
- Known-dangerous patterns (rm -rf, git push --force) hard-deny
- Unknown tools/patterns → ask
- Unit tests cover all presets and edge cases
