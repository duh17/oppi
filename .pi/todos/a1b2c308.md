## Problem (Review #3)

`PermissionCardView.resolve` uses `try?` which silently swallows WebSocket errors. If the connection is down when the user taps Allow/Deny, `isResolving` stays `true` forever — button appears stuck.

## Fix

Catch the error, reset `isResolving = false`, show inline error or toast so the user knows to retry.

## File

`ios/PiRemote/Features/Chat/PermissionCardView.swift`

## Priority

Fix before first device test — permission approval is the money feature.
