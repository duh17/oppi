## Context

Oppi is intentionally developer-focused and simple, but skill curation/review is a
core workflow. We need better visibility into where skills were used during a
session so users can review outputs and decide what to keep/promote.

Pi RPC does not emit explicit `skill_start`/`skill_end` events, so grouping needs
to be derived from existing messages/events.

Tracker: TODO-992ad1a6

## What to do

### V1 (iOS-only, heuristic grouping)

1. Detect likely skill invocations from existing data:
   - outgoing prompt starts with `/skill:<name>`
   - trace replay user messages containing `<skill name="...">` blocks
   - optional inferred signal: tool args referencing `.../skills/<name>/SKILL.md`

2. Add grouped UI sections in chat timeline:
   - collapsible "Skill run" row
   - header: skill name, tool count, duration estimate
   - expanded: existing tool rows + assistant output (reuse current rendering)

3. Add session outline support:
   - new filter chip: `Skills`
   - rows show concise summary per skill run

4. Keep it low-risk:
   - no protocol changes required for V1
   - unknown/unmatched cases fall back to current ungrouped timeline

### V2 (optional accuracy upgrade)

- Forward `turn_start` / `turn_end` through pi-remote to iOS and bind skill groups
to turn boundaries instead of heuristics.
