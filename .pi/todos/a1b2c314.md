## Summary (NNW §1.4)

Two related UX issues around stopping sessions:

1. **Duplicate stop taps**: If user taps "Stop" while a stop is already in-flight, a second request fires. Add idempotent UI lock (disable button while in-flight).
2. **Terminal state banner**: After a forced stop, there's no clear visual indicator that the session has ended. Add an explicit banner.
3. **Server reconciliation**: After stop timeout, auto-send `get_state` to reconcile.
