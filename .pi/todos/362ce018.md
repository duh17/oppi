## Context

Agent saves output to /work but there's no API to access those files.
The iOS app needs endpoints to browse and download workspace files.

Tracker: TODO-992ad1a6

## What to do

### A. Create files.ts module

New `pi-remote/src/files.ts` with:

- `listFiles(baseDir, subpath?)` → `{ name, path, size, modified, isDir }[]`
- `resolveAndValidate(baseDir, requestedPath)` → resolved path or throw
  - `realpath()` all paths, verify within baseDir
  - Block `..` traversal and symlinks outside workspace
- `contentType(filepath)` → appropriate Content-Type header
  - .md → text/markdown, .html → text/html, .json → application/json
  - .png/.jpg/.gif/.svg → image/*, .pdf → application/pdf
  - .py/.ts/.js → text/plain, .csv → text/csv
  - default → application/octet-stream
- Size limit: reject serving files > 50MB

### B. Add endpoints to server.ts

Two new routes (authenticated, session ownership verified):

```
GET /sessions/:id/files?path=/subdir
  → { files: [{ name, path, size, modified, isDir }] }
  Default path: / (workspace root)

GET /sessions/:id/files/*path
  → file contents with Content-Type header
  → 404 if not found, 403 if outside workspace
```

Use `sandbox.getWorkDir(userId, sessionId)` as the base directory.

### C. Path security

- All paths through `resolveAndValidate()` before any read
- Return 403 for any path that resolves outside workspace dir
- No directory listing via the content endpoint (only via the list endpoint)
