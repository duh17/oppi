## Context

Dedicated session types that seed the workspace and system prompt for
building or improving skills.

Depends on: TODO-ba1373dd (system prompt), TODO-eabe2bf3 (skill storage),
  TODO-d6c60004 (load user skills)
Tracker: TODO-992ad1a6

## What to do

### A. Session purpose field

Add `purpose` to CreateSessionRequest and Session in types.ts:

```typescript
interface CreateSessionRequest {
  name?: string;
  model?: string;
  purpose?: "general" | "skill-creation" | "skill-refinement";
  skillRef?: { name: string };  // for refinement: which skill
}
```

### B. Skill-creation session setup

When `purpose: "skill-creation"`, in initSession():

1. Seed workspace with skill template:
   ```
   /work/<name>/
   ├── SKILL.md      (template with frontmatter skeleton)
   └── scripts/      (empty dir)
   ```

2. Bundle skill-creation references into session (read-only):
   - Copy Anthropic's skill-creator SKILL.md to
     `<sessionDir>/references/skill-creation/`
   - Include 2-3 example skills for reference

3. Append to system prompt:
   ```
   ## Task: Build a new skill

   The user wants you to create a skill. Read the skill-creation guide
   at /home/pi/.pi/references/skill-creation/ before starting.
   Build the skill in /work/<name>/. It must have a SKILL.md with
   frontmatter (name, description) and optionally scripts/.
   ```

### C. Skill-refinement session setup

When `purpose: "skill-refinement"` with `skillRef: { name }`:

1. Copy existing skill into workspace:
   ```
   /work/<name>/     ← full copy of current skill
   ```

2. Append to system prompt:
   ```
   ## Task: Improve an existing skill

   The user wants to refine the skill '<name>'. The current version
   is in /work/<name>/. Read SKILL.md to understand what it does,
   then wait for the user's instructions.
   ```

### D. Wire POST /sessions

Update session creation in server.ts / sessions.ts to pass purpose
and skillRef to sandbox.initSession().

## Files

- `pi-remote/src/types.ts` — add purpose, skillRef to session types
- `pi-remote/src/sandbox.ts` — workspace seeding + prompt appendix
- `pi-remote/src/server.ts` — pass new fields through
- `pi-remote/sandbox/references/` — bundled skill-creator guide + examples

## Done when

- `POST /sessions { purpose: "skill-creation", name: "data-viz" }`
  creates session with template in /work/data-viz/
- `POST /sessions { purpose: "skill-refinement", skillRef: { name: "data-viz" } }`
  creates session with existing skill in /work/data-viz/
- Agent in creation session reads skill-creator reference unprompted
- Agent in refinement session reads existing SKILL.md and waits for input
