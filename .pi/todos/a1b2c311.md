## Fixed

`ServerCredentials.baseURL` and `webSocketURL(sessionId:)` already return `URL?` (not force-unwrapped). `ServerConnection.configure()` guards with `guard let baseURL = credentials.baseURL else { return false }`. `WebSocketClient.openWebSocket()` guards with `guard let url = credentials.webSocketURL(sessionId:) else { disconnect() }`.

No production force-unwraps, `fatalError`, `try!`, or `as!` remain in the iOS target.

Confirmed fixed as part of TODO-745032b2.
