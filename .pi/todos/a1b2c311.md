## Problem (Review #12)

`ServerCredentials.baseURL` force-unwraps `URL(string:)`. If `host` contains spaces or special characters (corrupted QR code), the app crashes.

## Fix

Guard with optional binding, or validate the host string during QR code parsing. Return a sensible error instead of crashing.

## File

`ios/PiRemote/Core/Models/User.swift`

## Priority

Fix before first device test — crash on bad QR.
