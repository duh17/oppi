## Context

Agent doesn't know it's in a container, what tools it has, or that the user
is on a phone. Needs a system prompt that shapes behavior for mobile consumption.

Depends on: TODO-52f6481d (skill sync — so referenced skills actually exist)
Tracker: TODO-992ad1a6

## What to do

### A. Create system-prompt.md template

Create `pi-remote/sandbox/system-prompt.md` with three sections:

**Environment** (what's available):
- Working dir `/work`, pi state at `~/.pi`, container is Alpine + Node + Python
- CLI tools on PATH: `search`, `fetch`, `fetch --browser`
- Standard dev tools: git, rg, fd, jq, make, tree, sqlite, uv
- Skills available: list dynamically based on what's installed

**Mobile output contract** (how to behave — this is the critical part):
- Work autonomously. Don't ask for confirmation on intermediate steps.
- Save substantial output to files in /work. Never dump >20 lines inline.
- End tasks with structured summary: what you did, files created, what
  needs attention.
- For errors, show key message inline, save full trace to file.
- Use brief progress markers for multi-step work.
- User can browse /work files on their phone — reference files by path.

**Permission gate** (what to expect):
- Some tool calls need phone approval. Wait for response, don't retry.
- The user sees the request on their phone and will approve/deny.

### B. Generate prompt dynamically in initSession()

In `sandbox.ts` `initSession()`, generate the prompt file at
`<sessionDir>/system-prompt.md` using template literals:

- List only actually-installed skills (don't mention web-browser if Chromium
  isn't available)
- Include user name for personalization
- Include model name (affects context window advice)

### C. Mount and pass to pi in spawnPi()

```typescript
args.push("-v", `${realpath(promptPath)}:/tmp/system-prompt.md:ro`);
// ... after image name:
piArgs.push("--append-system-prompt", "/tmp/system-prompt.md");
```

## Files
