## Context

When the agent writes files to /work, the user should see tappable file
cards in the chat timeline that link to the file browser/preview.

Depends on: TODO-15956340 (file browser), TODO-cf1dbddf (preview views)
Tracker: TODO-992ad1a6

## What to do

### A. Detect file paths from tool calls

In `TimelineReducer.swift`, when processing `tool_end` events for `write`
or `edit` tools:

1. Extract the `path` argument from tool_start args
2. Check if path is under /work (ignore internal files)
3. Attach file reference to the next assistant message's metadata

Model addition to ChatItem:
```swift
struct FileReference {
    let path: String        // "/work/report.md"
    let displayPath: String // "report.md"
    let tool: String        // "write" or "edit"
}
```

### B. FileChipView

New view rendered inline in assistant message bubbles:

```
┌──────────────────────────┐
│ 📄 report.md     8.4 KB │
│ [Tap to preview]         │
└──────────────────────────┘
```

- Icon based on file extension (same SF Symbols as FileRowView)
- Tapping navigates to FilePreviewView for that path
- Styled as a card inside the message bubble (muted background)

### C. Wire into ChatBubbleView

After the message text content, render any attached FileReferences
as FileChipView instances. Tapping opens the preview in a sheet or
navigation push.

## Files

- `ios/PiRemote/Core/Runtime/TimelineReducer.swift` — detect file paths
- `ios/PiRemote/Core/Runtime/ChatItem.swift` — add FileReference
- `ios/PiRemote/Features/Chat/FileChipView.swift` — NEW
- `ios/PiRemote/Features/Chat/ChatBubbleView.swift` — render file chips

## Done when

- Agent writes /work/report.md → chat shows tappable "report.md" card
- Tapping the card opens the file preview
- Only /work paths shown (internal tool files ignored)
