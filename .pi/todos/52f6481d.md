## Context

Pi-remote sessions can code but can't search the web, fetch docs, or browse.
The container image is slim and `initSession()` installs zero skills.

Tracker: TODO-992ad1a6

## What to do

### A. Fatten the container image

Edit `pi-remote/sandbox/Containerfile`:

```dockerfile
# Content extraction (for fetch skill)
RUN npm install -g readability-cli

# Headless Chromium (for web-browser skill)
RUN apk add --no-cache \
    chromium nss freetype harfbuzz ttf-freefont font-noto-emoji
ENV CHROME_BIN=/usr/bin/chromium-browser
ENV CHROME_PATH=/usr/lib/chromium/

# Python research tools
RUN pip install --no-cache-dir --break-system-packages \
    beautifulsoup4 markdownify trafilatura httpx
```

Rebuild and verify: `container build`, then `container run` + test each tool.

### B. Sync built-in skills in initSession()

In `pi-remote/src/sandbox.ts`, after the permission-gate install:

1. Define built-in skill list:
   ```
   searxng, fetch, web-browser, research-report, news-curator
   ```
2. For each skill, copy from host `~/.pi/agent/skills/<name>` into
   `<sessionDir>/agent/skills/<name>/` using `cpSync` with `dereference: true`
   (host skills are symlinks into dotfiles)
3. Fall back to bundled copies in `pi-remote/skills/` if host skill missing
4. Use same `isNewer()` pattern as auth.json for idempotent re-sync

### C. Create shim binaries

In `initSession()`, create `<sessionDir>/bin/` and symlink:
```
bin/search  → ../agent/skills/searxng/scripts/search
bin/fetch   → ../agent/skills/fetch/scripts/fetch
bin/fetch-allow → ../agent/skills/fetch/scripts/fetch-allow
```

Container already has `ENV PATH="/home/pi/.pi/bin:${PATH}"`.

### D. Create default fetch allowlist

Write a default `<sessionDir>/.config/fetch/allowed_domains.txt` with common
doc domains (github.com, docs.python.org, developer.mozilla.org, etc.) so
fetch works out of the box without syncing personal allowlist.

## Files

- `pi-remote/sandbox/Containerfile` — add deps
- `pi-remote/src/sandbox.ts` — initSession(): skill sync + shim creation
- `pi-remote/skills/` — bundled fallback copies (for fresh installs)

## Done when

- `container run pi-remote:local -- which readability-cli` succeeds
- `container run pi-remote:local -- chromium-browser --version` succeeds
- New session has skills in agent/skills/ and shims in bin/
- Agent can `search "test"` and `fetch "https://example.com"` in a session
