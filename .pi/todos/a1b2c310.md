## Problem (Review #7)

The `isNearBottom` sentinel in `ChatView` uses `onAppear`/`onDisappear` of a 1pt `Color.clear` in a `LazyVStack`. On initial load of a busy session, the sentinel was never visible, so `isNearBottom` stays `false` and auto-scroll is disabled. User sees the top of the conversation instead of the bottom.

## Fix

Scroll to bottom on initial load in `connectToSession()` after `loadFromREST` completes. `isNearBottom` is already initialized to `true`, but an explicit initial scroll ensures the user lands at the bottom.

## File

`ios/PiRemote/Features/Chat/ChatView.swift`

## Priority

Fix before first device test — first thing user sees.
