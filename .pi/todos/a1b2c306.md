## Summary

Simple CLI script that connects via WebSocket, prints permission requests, and lets the operator approve/deny with keyboard input. Validates the full end-to-end flow before building the iOS app.

## Design

- Connect to server WebSocket with bearer token
- Print incoming `permission_request` messages with tool name, risk level, command
- Accept keyboard input: `y` (allow), `n` (deny), `a` (always allow this pattern)
- Send `permission_response` back

## Scope

- New file: `pi-remote/tools/test-client.ts` (or similar)

- v1 Step 6 (final validation)
- Depends on: a1b2c305

## Acceptance

- Full flow works: pi spawns → extension loads → tool intercepted → request shown → user approves → tool executes
- Denied tool calls return `{ block: true }` to pi
- Timeout after 2 minutes returns deny
