## Summary

Create the pi extension at `extensions/permission-gate/index.ts` that intercepts tool calls and routes them through the gate socket.

## Design

- Connects to Unix socket via `net.createConnection()` using `PI_REMOTE_GATE_SOCK` env var
- `before_agent_start` → sends `guard_ready`
- `tool_call` → sends `gate_check`, blocks on response
- Heartbeat timer every 15s
- On socket disconnect → block all tool calls (fail-safe)

## Scope

- File: `pi-remote/extensions/permission-gate/index.ts`
- File: `pi-remote/extensions/permission-gate/package.json`

- v1 Step 4

## Acceptance

- Extension connects to gate socket on startup
- `guard_ready` sent successfully
- Tool calls blocked until gate response received
- Heartbeat keeps connection alive
- Socket errors → all tool calls blocked (not silently passed)
- Test: run pi with extension locally, verify tool calls go through socket
