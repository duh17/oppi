## Summary

Connect the gate server, extension, and WebSocket layer so the full permission flow works end-to-end.

## Changes

1. **sessions.ts**: On spawn, create gate socket, pass socket path via env, install extension into sandbox
2. **server.ts**: Forward `approval_needed` events to phone's WebSocket as `permission_request`
3. **server.ts**: Handle `permission_response` from phone WebSocket, call `gate.resolveDecision()`
4. **types.ts**: Add `permission_request` and `permission_response` to WS message types

## Session State Machine

```
STARTING → SPAWNED → GUARDED → READY ↔ BUSY → STOPPED
              ↓         ↓                         ↑
           (no guard_ready within 10s)            │
              ↓         ↓                         │
           FAILED ←── FAIL_SAFE ──────────────────┘
```

## Acceptance

- Guarded session handshake works (guard_ready → guard_ack within 10s)
- Permission requests forwarded to connected WebSocket clients
- Permission responses route back to gate and unblock extension
- FAIL_SAFE mode entered on heartbeat loss
