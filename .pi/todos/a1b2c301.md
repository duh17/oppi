## Problem

`waitForReady()` in `sessions.ts:114-130` creates a second readline interface on `proc.stdout` that competes with the main one. The first message gets misrouted to the `unknown/<sessionId>` key.

## Fix

Replace dual-readline with a single readline consumer that routes messages based on content. Only one readline instance should exist per process stdout.

## Scope

- File: `pi-remote/src/sessions.ts`
- ~30 min estimate
- v1 Step 1 (prerequisite for everything else)

## Acceptance

- Single readline consumer per process
- First message correctly routed
- Existing session tests still pass
