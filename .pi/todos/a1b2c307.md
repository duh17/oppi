## Problem (Review #2)

If the user quickly navigates back and forward to the same session, `.task(id: sessionId)` won't re-fire because the id didn't change. The stream is gone (disconnected in `onDisappear`) but the task won't restart.

## Fix

Add `@State var connectionGeneration: Int = 0`, increment in `onAppear`, use as the task id instead of (or combined with) `sessionId`.

## File

`ios/PiRemote/Features/Chat/ChatView.swift`

## Priority

Fix before first device test — will hit this in real use immediately.
