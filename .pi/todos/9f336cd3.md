## Goal
When a skill is created/edited/deleted on the host, automatically re-sync it into active container workspaces that use that skill.

## Flow
1. FSWatcher detects skill change → `skills:changed` event fires
2. For each active container workspace using the changed skill:
   - Re-copy the skill dir into the workspace sandbox
   - Rebuild bin shims if scripts changed
3. New sessions in that workspace get the updated skill

## Key Decisions
- **Active sessions**: Don't disrupt. Pi already loaded the skill into context. The updated skill will be available on the next `read` of SKILL.md, or on next session.
- **Granularity**: Only re-sync the specific skill(s) that changed, not the whole skills dir
- **Debounce**: Batch changes within 1s window to avoid thrashing

## Files
- `src/sandbox-skills.ts` — add `resyncSkill(workspaceRoot, skillName)` 
- `src/sandbox.ts` — listen to `skills:changed`, trigger re-sync for affected workspaces
- `src/skills.ts` — emit event with changed skill names
