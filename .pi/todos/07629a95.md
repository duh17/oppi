## Problem

`sandbox.ts` syncs `auth.json` from host `~/.pi/agent/auth.json` only at session creation. Anthropic OAuth access tokens are short-lived (~8h). When they expire mid-session, the sandboxed pi agent loses API access.

## Observed

- Session `PgRegYlX` (research) lost Anthropic access after ~2h because its `auth.json` copy had an expired token.
- Host's `auth.json` had a fresh token (refreshed by interactive pi usage).
- Manual fix: `cp ~/.pi/agent/auth.json ~/.pi-remote/sandboxes/<user>/<session>/agent/auth.json`

## Fix Options

1. **Re-sync on each prompt call** — `sessions.ts` copies host auth.json before sending prompt to pi RPC. Simple, reliable.
2. **Periodic background sync** — Timer in `sandbox.ts` re-syncs every N minutes for all running sessions.
3. **Watch file** — `fs.watch` on host auth.json, sync on change.

Option 1 is simplest and sufficient.

## Resolution\n\nSuperseded by auth proxy (TODO-5d327779). With the proxy injecting credentials at request time, tokens never enter the container and refresh is handled host-side. No per-session re-sync needed.
