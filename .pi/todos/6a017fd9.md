## Goal
Allow changing a workspace's skill list without destroying/recreating the workspace.

## Changes
- `PATCH /workspaces/:id` already exists — ensure `skills` field is mutable
- When skills change on a live workspace:
  - **Container mode**: re-sync skills dir into the workspace sandbox root
  - **Host mode**: no-op (pi discovers from host skill dirs directly)
- New sessions in the workspace pick up the updated skill list
- Active sessions keep their current skills (no mid-session disruption)

## API
```
PATCH /workspaces/:id
{ "skills": ["search", "fetch", "my-custom-skill"] }
```

## Container Re-sync Flow
1. Workspace skill list changes
2. Diff: added skills → copy into workspace skills/
3. Diff: removed skills → delete from workspace skills/
4. Rebuild bin shims for new scripts
5. Next session launch gets the updated skills
