## Goal

Browse skills loaded in a workspace from the iOS app. View SKILL.md content, file trees, and start a session to improve a skill.

## Completed

### Phase 1: Server endpoints (done)

**`GET /skills/:name`** — returns `{ skill, content, files }`:
- `skill`: SkillInfo (name, description, containerSafe, hasScripts, path)
- `content`: raw SKILL.md markdown
- `files`: flat file tree (excludes __pycache__, .pyc, node_modules, etc.)

**`GET /skills/:name/file?path=scripts/search.py`** — returns `{ content }`:
- Path traversal guarded via realpath + startsWith
- 1MB file size limit

Files modified:
- `pi-remote/src/skills.ts` — added `SkillDetail` type, `getDetail()`, `getFileContent()`, `listFiles()` methods
- `pi-remote/src/server.ts` — added route matching + handlers for both endpoints

### Phase 2: iOS SkillDetailView (done)

New files:
- `ios/PiRemote/Core/Models/SkillDetail.swift` — `SkillDetail` model (skill + content + files)
- `ios/PiRemote/Features/Skills/SkillDetailView.swift` — SKILL.md viewer with metadata badges, markdown rendering, file tree
- `ios/PiRemote/Features/Skills/SkillFileView.swift` — individual file viewer (markdown or monospaced code)

Modified:
- `ios/PiRemote/Core/Networking/APIClient.swift` — `getSkillDetail(name:)`, `getSkillFile(name:path:)`
- `ios/PiRemote/Core/Services/TimelineCache.swift` — `loadSkillDetail()`, `saveSkillDetail()` (stale-while-revalidate)
- `ios/PiRemote/Features/Workspaces/WorkspaceEditView.swift` — skill rows now have disclosure chevron → SkillDetailView

Navigation: `SkillDetailDestination` + `SkillFileDestination` Hashable types for type-safe `navigationDestination`.

All 556 iOS tests pass. Server tsc clean. iOS build clean.
