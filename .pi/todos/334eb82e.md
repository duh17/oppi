## Goal
Make SkillRegistry reactive — watch skill directories for changes and re-scan automatically.

## Changes
- Add `fs.watch` (recursive) on all scan dirs (`~/.pi/agent/skills/`, `~/.config/oppi/skills/`, bundled)
- Debounce re-scan (500ms) to handle rapid file changes
- Emit `skills:changed` event (EventEmitter) so other systems can react
- Log what changed: added/removed/modified skills
- `registry.scan()` becomes safe to call anytime (idempotent, fast)

## Files
- `src/skills.ts` — add watcher + EventEmitter to SkillRegistry

## Testing
- Unit test: mock fs events → verify re-scan fires
- Manual: edit a SKILL.md → see log line within 1s

## Notes
- macOS `fs.watch` recursive works natively (no chokidar needed)
- Don't watch inside node_modules/__pycache__/etc
- Container compat: move from heuristic string matching to explicit `container: true/false` in SKILL.md frontmatter (fall back to heuristics if missing)
