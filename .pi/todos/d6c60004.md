## Context

TODO-52f6481d syncs built-in skills. This TODO adds user-created skills
to the mix so they're available in every new session.

## What to do

In `sandbox.ts` `initSession()`, after syncing built-in skills:

1. Read user's skill directory from `~/.pi-remote/skills/<userId>/`
2. For each skill directory, copy into `<sessionDir>/agent/skills/<name>/`
   using cpSync with dereference
3. Skip any that collide with built-in names (shouldn't happen due to
   save validation, but defensive)
4. Update system prompt generation to include user skills in the
   "available skills" list

Merge order:
```
1. Built-in skills (searxng, fetch, web-browser, etc.)
2. User skills (from ~/.pi-remote/skills/<userId>/)
```

User skills that have scripts/ get shim binaries created in bin/ just
like built-ins do.
