## What

Add workspace support to the iOS app:

1. **Workspace picker** — shown when tapping "New Session". Grid of workspace cards (icon + name + description). Tapping a workspace creates a session with that workspaceId.

2. **Workspace management** — Settings → Workspaces. List view with tap-to-edit:
   - Name, description, icon
   - Skills: toggleable chips from GET /skills pool (show ⚠️ badge for non-container-safe)
   - Policy preset: segmented control (Container / Restricted)
   - Host mount: text field (validated server-side)
   - Default model: picker from /models list
   - System prompt: text editor

3. **Session list** — show workspace name/icon badge per session

4. **API client** — add methods:
   - `listWorkspaces() -> [Workspace]`
   - `createWorkspace(req) -> Workspace`
   - `updateWorkspace(id, req) -> Workspace`
   - `deleteWorkspace(id)`
   - `listSkills() -> [SkillInfo]`

## Server API (already implemented)

```
GET    /workspaces
POST   /workspaces
GET    /workspaces/:id
PUT    /workspaces/:id
DELETE /workspaces/:id
GET    /skills
POST   /skills/rescan
```

Session creation: `POST /sessions { workspaceId: "..." }`
