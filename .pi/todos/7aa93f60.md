## Context

We want coding-CLI style input ergonomics in Oppi chat:
- `/command` discovery + insertion
- `@file` filename/path completion

Current state:
- Slash execution already works through existing prompt flow (`prompt("/...")`) and `get_commands` RPC already exists end-to-end.
- iOS has no slash/autocomplete UI yet.
- `@file` in pi interactive mode is primarily autocomplete + token insertion UX. RPC mode does not currently provide iOS-friendly file suggestion APIs.

## TODO

- [x] Define slash trigger rules (`/` at message start vs anywhere)
  - Decision: trigger only when current token starts at message start (`/foo`), not mid-sentence.
- [ ] Define `@file` trigger rules (token boundaries, quoting, spaces)

## Acceptance criteria

- [x] Typing `/` shows available commands (RPC + built-in), selecting one inserts correctly, send works.

## Delivery plan

1. Ship slash command autocomplete first (client-only, quickest path).
2. Add server file-suggest API.
3. Ship `@file` autocomplete using that API.
