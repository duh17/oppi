## Summary (NNW §1.5)

Remaining `fatalError` / force-unwrap crash paths in the app layer need to be replaced with graceful error handling.

## Known Locations

- `Core/Models/User.swift` — `fatalError` paths
- `ServerCredentials.baseURL` force-unwrap (also tracked in a1b2c311)
- Any other `!` in non-test Swift files

## Approach

- Audit all Swift files for `!` (excluding tests and IB outlets)
- Replace with optional binding, guard-let, or meaningful error states
- Goal: "No crashes" (NNW reliability value #2)
